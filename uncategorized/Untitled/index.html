(五)、“insert/update”注入(报错注入)

​ ①先了解一下什么是报错注入：

​ 报错注入就好像你试图通过一些特殊的手段让一个程序出错，然后利用程序报错时返回的信息，来获取你想要的一些秘密。注册页面如果有注入漏洞的话，一般是insert类型的，因为注册相当于往数据库的表中插入一行新数据

​ ★☆注意：此类型注入在实际测试中尽量不要使用，操作不当可能会导致数据库插入大量脏数据！！！

​ ②首先在注册功能点输入任意内容，填写注册信息，然后抓个包，可以看到时post形式传输的数据，并在最后一个参数后面加入单引号，发现出现了sql语句报错

​ ☆注意 这种insert/update类型的sql在进行注入时尽量在最后一个参数插入payload进行注入，这样可以更容易闭合sql语句完成注入，

​ 像本关的这个insert语句我们直接就在住址(Add)这个参数进行注入，闭合前面的sql只需要加入’$payload)(前后分别加入单引号和右括号)

​ 查询user表的列

​ 并没有查询完整，使用limit、substr函数，limit之前学过，了解一下substr:

  • string_expression: 要提取子字符串的原始字符串。
  • start: 提取子字符串的起始位置。如果为正数，则表示从字符串的开头开始计算位置；如果为负数，则表示从字符串的末尾开始计算位置。
  • length: 可选参数，指定要提取的子字符串的长度。如果未指定，则提取从起始位置到字符串末尾的所有字符。

​ 例如：

​ SELECT SUBSTRING(‘Hello, world!’, 1, 5);

​ 上述 SQL 查询将返回从字符串 'Hello, world!' 中索引位置 1 开始的 5 个字符，即 'Hello'。

​ 表示从开始字符截取往后所有的字符串
