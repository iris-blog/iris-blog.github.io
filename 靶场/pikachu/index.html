浏览器：火狐

使用前：打开burp，火狐burp代理，打开小皮面板的apache、mysql（不打开没法做）

<暴力破解>

一、基于表单的暴力破解

​ 拦截显示开启即为正常运行，打开火狐浏览器

​ 拦截成功后发送到Intruder

​ 先清除所有payload，若要爆破密码则在密码（自己随便填，选中块状区域），点击添加payload；username同理；

​ 再点击有效载荷，在payload选项里“从文件加载”选择字典，点击右上角开始攻击，点击长度，可出现不同长度的爆破值，所显示的值即为爆破的密码

​ （没图，储存桶删了，之前的o(╥﹏╥)o）

二、验证码绕过

​ 1、on server（后台会验证）

​ 抓包状态，随便输入账号密码，先不输入验证码

​ 发送到repeater

​ 打开repeater，点击send（英文版好像是GO），右侧会显示验证码不能为空

​ 随便输入验证码提示验证码错误

​ 然后输入正确验证码，提示账号密码错误（说明会交到后台，是on server类型，其实也可以查看网页源代码）

​ 接下来吼吼暴力破解了

​ 还是老样子，清除，添加，不同的是添加账号跟密码一起，同时攻击类型要选集束炸弹

​ 然后payload set分别选择1、2，分别在payload选择里加载用户名字典、密码字典，进行爆破。

​ 俺第一次用户名字典忒多了，2万多，睡了两个小时也没弄完，简化后有两个

​ test abc123跟admin 123456，都试了一下(试的时候别忘了把burp关了，不然到死都不放行(￣ω￣;))，后者对，完毕

​ 2、on client（后台不验证，★必须输入正确验证码★，否则不可提交）

​ 输入错误验证码，提示错误糙级快，立马就弹出错误嘞，打开f12发现没有http请求，那就说明验证码从前端JS来噜。

​ 确认后台没有认证：

①发送带repeater，send，显示账户或密码错误

②删除验证码或修改，继续send，仍然是账号密码错误

③确认无认证

​ 放行几次才有正常页面呢

​ 发送intruder，重复1、on server 的步骤，添加账户密码变量放入字典爆破

​

​ 仍然是俩，挨个试试
