一、SQL注入简介

​ ☆SQL注入的成因：

​ 开发人员在开发过程中，将URL的参数，http body 中post参数以及外来的用户输入（例如cookies）与SQL语句拼接，使得待执行的SQL语句可控；

​ ☆分类：

可回显的注入： 联合查询注入、报错注入、通过注入进行DNS请求
不可回显的注入： Bool盲注（返回true、false和报错的语句）、时间盲注
二次注入： 一般需要自己编写脚本

​ ☆字符串字面量必须用单引号括起来，此时单引号不起到单引号的作用，如果要用到单引号，那就需要连续两个单引号表示（单引号转义）。

注意：数字不需要引号，两个中杠–在SQL中表示注释，–后面的部分是被注释的部分，不起作用

​ ☆针对数值的SQL注入攻击，当出现非数值的字符时就会视为数值字面量终止，比如下图，1后加了一个分号，那么分号后的值被解释为SQL语句的一部分

2、字符串类型的查询

​ 相关语法规则：

Operator（操作符） Condition（解释） Example（例子）
= 完全等于 col_name = “abc”
!= or <> 不等于 col_name != “abcd”
LIKE 没有用通配符等价于 = col_name LIKE “ABC”

% 通配符，代表匹配0个以上的字符 col_name LIKE “%AT%”

_(下滑线) 和% 相似，代表1个字符 col_name LIKE “AN_”
(matches “AND”, but not “AN”)
IN(……) 在列表 col_name IN (“A”, “B”, “C”)
NOT IN(……) 不在列表 col_name NOT IN (“D”, “E”, “F”)

​ 1、DISTINCT 语法介绍

​ 我们拿之前的 Movies表来说，可能很多电影都是同一年Year发布的，如果你想要按年份排重，一年只能出现一部电影到结果中， 你可以用 DISTINCT 关键字来指定某个或某些属性列唯一返回。写作：DISTINCT Year

​ ☆因为DISTINCT 语法会直接删除重复的行, 还要学习 GROUP BY 语句， GROUP BY 也会返回唯一的行，不过可以对具有相同的 属性值的行做一些统计计算(例如：求和)

​ 选取出唯一的结果的语法

​ 2、结果排序

​ 在实际的数据表中，数据添加的时候不是完全顺序的，为了解决结果排序问题, 我们可以用 ORDER BY col_name 排序的语法来让结果按一个或多个属性列做排序.

​ 基本结构： （ASC升序 或 DESC 降序，对数字来说就是升序 1，2，3，… 或降序 … 3,2,1 . 对于文本列，升序和降序指的是按文本的字母序。）

​ ☆ORDER BY col_name 这句话的意思就是让结果按照 col_name 列的具体值做 ASC升序 或 DESC 降序

​ 3、通过Limit选取部分结果（Limited 查询）

六、用JOINs进行多表联合查询

​ 我们已经学会了如何做一个单表的查询，但是在现实数据库中往往包含一组相关的数据表，这些表一般会符合数据库范式。 让我们先了解下关系数据库的范式

​ 1、数据库范式

​ 数据库范式是数据表设计的规范，在范式规范下，数据库里每个表存储的重复数据降到最少（这有助于数据的一致性维护），同时在数据库范式下，表和表之间不再有很强的数据耦合，可以独立的增长。

​ 如果一个实体（比如Dog）的属性数据被分散到多个数据表中，我们就需要学习如何通过 JOIN连表技术来整合这些数据并找到我们想要查询的数据项

​ 2、用JOINs进行多表联合查询

​ ①主键：一般关系数据表中，都会有一个属性列设置为主键；唯一标识一条数据，不会重复，比如身份证号，最常见的是自增ID，每写入一行数据ID+1, 当然字符串，hash值等只要是每条数据是唯一的也可以设为主键.

​ ②主键作用：可以把两个表中有相同主键ID的数据连接（因为一个ID可以简要的识别一条数据，所以连接之后还是表达的同一条数据）

​ ③JOIN关键字：INNER JION
